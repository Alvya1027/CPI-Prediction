# 孪生网络与对比学习笔记

> 本文档面向项目组全体成员。第5阶段需要构造"时间片段对"并用孪生结构进行对比学习，这里先把基础概念和策略讲清楚。

---

## 1. 什么是孪生网络（Siamese Network）

### 1.1 一句话解释

孪生网络由两个共享权重的相同子网络组成，输入两个样本，输出它们是否"相似"。两个子网络完全相同，参数共享——这就是"孪生"的含义。

### 1.2 架构示意

```
输入A → [Encoder A] → 向量A ─┐
        (与B共享权重)          ├→ 距离度量 → 相似/不相似
输入B → [Encoder B] → 向量B ─┘
```

### 1.3 共享权重是什么意思

Encoder A 和 Encoder B 是同一个网络。训练时，两个样本的梯度会累加后一起更新这一份权重。这保证了两个样本被映射到同一个特征空间，在这个空间中可以直接比较它们的距离。

## 2. 对比学习（Contrastive Learning）

### 2.1 核心思想

对比学习的目标是：让相似的样本在特征空间中靠近，让不相似的样本远离。

这和传统分类不同——分类只关心"这是什么"，对比学习关心的是"它们像不像"。

### 2.2 Contrastive Loss

最经典的对比损失函数：

$$L = \frac{1}{2N} \sum_{i=1}^{N} \left[ y_i \cdot d_i^2 + (1 - y_i) \cdot \max(0, m - d_i)^2 \right]$$

其中：
- $d_i = \|f(A_i) - f(B_i)\|_2$：两个样本在特征空间的欧氏距离
- $y_i$：标签，1 表示相似，0 表示不相似
- $m$：margin 参数，不相似样本对的距离超过 $m$ 时不再惩罚
- $f(\cdot)$：Encoder 输出

**直观理解**：
- 相似样本对（$y=1$）：最小化 $d^2$，把它们拉近
- 不相似样本对（$y=0$）：如果 $d < m$，就把它们推开；如果 $d \ge m$，说明已经够远了，不管

### 2.3 其他常见损失

| 损失函数 | 特点 | 适用场景 |
|----------|------|----------|
| Contrastive Loss | 需要明确的正/负标签 | 有监督对比 |
| Triplet Loss | 三元组（锚点+正例+负例） | 人脸识别等 |
| InfoNCE | 无需显式标签，自监督 | 大规模预训练 |
| SimCLR | 数据增强自生成正样本对 | 图像自监督 |

本项目第5阶段建议先用 Contrastive Loss，因为我们可以根据 CPI 的实际走势构造正/负样本对标签。

## 3. 时间序列如何构造相似片段

### 3.1 问题

CPI 是一个一维时间序列，怎么构造"相似片段对"和"不相似片段对"？

### 3.2 策略一：基于趋势方向

**正样本对（相似）**：选取两个 CPI 走势方向相同的片段。

```
示例：2008-01~2008-12（先升后降）和 2011-01~2011-12（先升后降）
→ 这两个片段趋势相似，标记为 y=1
```

**负样本对（不相似）**：选取走势方向相反或无关的片段。

```
示例：2008-01~2008-12（先升后降）和 2009-06~2010-06（持续上升）
→ 这两个片段趋势不同，标记为 y=0
```

### 3.3 策略二：基于数值区间

**正样本对**：CPI 值落在相近区间的两个片段。

```
示例：CPI 在 2%~3% 区间的两个 12 个月片段
→ 标记为 y=1
```

**负样本对**：CPI 值落在不同区间的两个片段。

```
示例：一个 CPI 在 1% 以下（通缩风险），另一个在 5% 以上（高通胀）
→ 标记为 y=0
```

### 3.4 策略三：基于统计特征

计算每个片段的统计量（均值、方差、趋势斜率），然后根据这些统计量的相似度构造样本对。

```python
# 伪代码
def compute_segment_features(segment):
    return {
        'mean': np.mean(segment),
        'std': np.std(segment),
        'trend': np.polyfit(range(len(segment)), segment, 1)[0],  # 斜率
        'max_min_ratio': (np.max(segment) - np.min(segment)) / np.mean(segment)
    }

# 特征空间中的距离小于阈值 → 正样本对
# 特征空间中的距离大于阈值 → 负样本对
```

### 3.5 建议：CPI 预测场景的推荐策略

对于 CPI 预测，我推荐**策略一（趋势方向）作为主策略**，原因：

1. CPI 的趋势方向比绝对值更有预测意义——预测模型关心的是"接下来会涨还是跌"，而不是"当前 CPI 等于多少"
2. 趋势方向判断直观，不需要调阈值
3. 和项目的最终目标一致——孪生光储备池做的是小样本下的趋势预测

## 4. 孪生储备池架构（项目用）

### 4.1 整体架构

```
时间片段A → [储备池 Encoder] → 状态向量A ─┐
           (共享权重：相同储备池矩阵)         ├→ Contrastive Loss
时间片段B → [储备池 Encoder] → 状态向量B ─┘
```

### 4.2 与普通储备池的区别

- **普通储备池**：输入→储备池→线性读出→预测值，用 MSE 训练
- **孪生储备池**：两个片段→**同一个**储备池→两个状态向量→对比损失，训练的是让相似片段的状态向量接近

### 4.3 训练流程

1. 从 CPI 历史数据中提取时间片段（窗口大小 = 12）
2. 构造正负样本对（基于趋势方向）
3. 两个片段分别输入同一个储备池
4. 获取储备池最后时刻的状态向量
5. 计算 Contrastive Loss
6. 优化储备池的输出权重（或反馈参数）
7. 训练完成后，用储备池状态向量 + 线性回归做预测

### 4.4 为什么这对小样本有帮助

孪生结构本质上是一种数据增强——虽然我们只有一条 CPI 序列，但通过构造正负样本对，训练信号从"每个样本一个标签"变成了"每对样本一个相似度标签"，有效增加了训练信息量。
